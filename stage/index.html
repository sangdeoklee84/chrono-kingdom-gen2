설치 · 계정

  • Claude Code 설치 + 로그인 — npm install -g @anthropic-ai/claude-code (또는 데스크톱 앱). Anthropic 구독/결제 필요
  • git 설치 — 클론용, 클론 전에 먼저
  • Node.js 20.6+ — 방법 B/C·도구용 (--env-file)
  • Higgsfield 계정 — 가입·로그인, 이미지 생성은 크레딧/유료 필요할 수 있음

저장소 접근 권한

  • chrono-stage는 비공개 저장소
  • 관리자에게 협업자 초대를 받고 수락
  • git이 GitHub에 인증돼 있어야 클론됨 (HTTPS면 Personal Access Token)

⚠️ Windows에서 스크립트가 막히면(running scripts is disabled): powershell -ExecutionPolicy Bypass -File install.ps1

빠른 시작 — 1분 설치

두 방법 모두 아래 2단계로 준비됩니다. 설치 스크립트는 시스템을 바꾸지 않는 안전한 점검형(파일 복사·안내만)이라 부담 없이 실행하세요.

1

저장소 클론

안에 README.md(사용법)와 STAGE-BACKGROUNDS.md(화풍·구도 템플릿)가 들어 있습니다.

git clone https://github.com/sangdeoklee84/chrono-stage.git
cd chrono-stage
2
한 번만 실행

설치 스크립트 실행

필수 도구 점검 → .env 자동 생성 → 다음 단계 안내까지 한 번에 끝납니다. OS에 맞는 것만 실행하세요.

# Windows  (탐색기에서 install.ps1 우클릭 → "PowerShell로 실행" 도 가능)
./install.ps1

# macOS / Linux
bash install.sh

스크립트가 끝나면 화면 안내대로 방법 A(권장) 또는 방법 B로 진행합니다. ↓

방법 A · Higgsfield

권장 · 설치 불필요

자기 Higgsfield 계정을 Claude Code에 연결합니다. 기본 방법(.mcp.json)을 우선 쓰고, 안 되면 보조 방법.

1
기본 · .mcp.json (http)

프로젝트 MCP로 연결

클론한 chrono-stage 폴더엔 .mcp.json이 있어, 이 폴더에서 Claude Code를 열면 higgsfield 서버가 자동 등록됩니다(허용 프롬프트 승인). 그다음 /mcp → higgsfield → 브라우저 로그인 + Authorize → 다시 /mcp로 connected 확인.

+
추가 · claude.ai 관리형 커넥터

대안 연결

기본이 안 되거나 claude.ai 커넥터를 선호하면: /mcp → "claude.ai Higgsfield" 선택 → 로그인 + Authorize. 둘 중 하나만 쓰세요(동시 사용 시 도구 중복).

2

이미지 요청 — 그냥 말로

초원 드루이드 유적 스테이지 배경을 GPT Image 2 16:9로 만들어줘
3

마무리 — 투명 / 규격화

스테이지는 tools/crop-2048x1024.ps1로 2048×1024 규격화. 배경 제거는 하지 않습니다(꽉 찬 배경).

방법 B · Gemini / GPT 스크립트 (Higgsfield 없이)

Higgsfield 계정이 없어도 OK — Gemini 또는 GPT(OpenAI) API 키만으로 생성. Node 20.6+ 필요(--env-file).

1
최초 1회만

키 채우기 (.env)

설치 스크립트가 만든 .env에 쓰는 쪽 키만 넣으면 됩니다. (둘 다 결제 필요)

# 둘 중 쓰는 것만:
GEMINI_API_KEY=본인_키
OPENAI_API_KEY=본인_키
2

Gemini로 생성 / 편집

node --env-file=.env gemini/gen.mjs "프롬프트" out/결과.png
# 기존 이미지 편집: ... --in 원본.png
3

GPT(OpenAI)로 생성

node --env-file=.env openai/gen.mjs "프롬프트" out/결과.png --size 1536x1024

size: 1024x1024 / 1536x1024 / 1024x1536. 스테이지는 이후 crop-resize로 규격화.

⚠️ 이 경로는 투명배경 자동 컷아웃이 없음 — 마을 에셋은 단색 배경 생성 후 외부 도구로 컷아웃 필요(Higgsfield 경로는 자동).

🌐 API·CLI 없이? 웹에서 수동

키도 Claude Code도 없으면 브라우저 웹만으로도 됩니다. 쓰는 서비스(지피티 / 제미나이)에 맞춰 아래에서 골라 따라하세요.

🗂️ 먼저 · 규약 문서부터 챙기기 (사전작업)

프롬프트를 짜기 전에, 어떤 규약이 어느 문서에 있는지 알고 그 문서를 손에 넣는 게 먼저입니다. 화풍·구도·규격은 전부 아래 문서에 정의돼 있어요.

README.md
여기서 시작 — 전체 사용법, 생성 경로(A·B·D), 폴더 구조 안내.
STAGE-BACKGROUNDS.md
핵심 규약 전부 — 화풍 고정 키워드 · 플레이필드 규정(50% 컷·빈 레인) · 카메라/HUD 세이프존 · 구도 타입(벨트스크롤) · 바이옴별 색 · 규격화(2048×1024) · 변형 세트 · 납품 체크리스트.
style-anchors/
화풍 통일용 첨부 이미지 — stage_stylesheet.jpg 등. 생성 시 첨부하면 톤·구도가 똑같이 따라옵니다(image-to-image).
↓ 이 문서들을 손에 넣는 두 가지 방법 ↓

🅰️ Claude Code · git 있는 사람

  1. 위 빠른 시작대로 git clone
  2. 클론한 폴더 안에 README · STAGE-BACKGROUNDS.md · style-anchors가 모두 들어있음
  3. 그대로 열어 규약 확인 → 생성으로

🅱️ Claude/git 없이 ChatGPT·Gemini만

  1. 비공개 repo 협업자 초대 수락 (관리자에게 요청)
  2. GitHub 웹에서 chrono-stage 접속 → STAGE-BACKGROUNDS.md 열어 본문 전체 복사
  3. style-anchors/의 stage_stylesheet.jpg → Raw → 우클릭 저장(다운로드)
  4. 복사한 규약은 대화에 붙여넣고, 앵커 이미지는 첨부 → 아래처럼 생성
💬 ChatGPT · 🟢 Gemini — 실제 대화는 이렇게 흘러갑니다
🧑
밝고 깔끔한 카툰 2D 스테이지 배경 … (아래 복붙 프롬프트 전체)+ 가로(landscape)로 그려줘
🤖
좋아요, 초원 스테이지입니다 👇
↳ 보통 1536×1024로 나옴 → 아래 Photopea에서 2048×1024로 크롭(위 하늘을 더 잘라 하단 플레이 레인 살리기)
🧑
이 스타일·구도 그대로, 오브젝트만 사막으로 바꿔줘
🤖
네! 같은 톤의 사막 버전이에요 👇
↳ 같은 대화에서 이어서 요청하면 세트 화풍이 자동으로 통일됩니다 ✨ (또는 첫 결과·stage_stylesheet.jpg를 첨부)

⬇️ 더 정확히 하려면 아래 서비스별 안내 · 복붙 프롬프트 · 규격화를 그대로 따르세요.

💬 지피티 (ChatGPT · chatgpt.com)

  1. 로그인 — chatgpt.com 접속·로그인. (이미지 생성은 유료 플랜/크레딧이 필요할 수 있음)
  2. 프롬프트 입력 — 아래 "복붙 프롬프트"를 붙여넣고, 끝에 가로(landscape)로 그려줘를 덧붙입니다.
  3. 비율 주의 — ChatGPT엔 16:9·2:1 정밀 옵션이 없습니다. 가로로 요청하면 보통 1536×1024(3:2)로 나옵니다 → 그대로 받고 아래 규격화에서 크롭.
  4. 세트(여러 장) 화풍 통일 — 첫 결과가 좋으면 같은 대화에서 "이 스타일·구도 그대로 오브젝트만 ○○로 바꿔줘"라고 이어서 요청. 또는 style-anchors/stage_stylesheet.jpg를 첨부해 "이 스타일로"라고 지시.
  5. 다운로드 → 아래 "규격화(공통)"으로.

👍 간판·글자 렌더링이 비교적 정확. 한 번에 1장씩 생성.

🟢 제미나이 (Gemini · gemini.google.com)

  1. 로그인 — gemini.google.com 접속(더 세밀히 제어하려면 AI Studio). 이미지 생성 모델(Nano Banana 계열) 사용.
  2. 프롬프트 입력 — 아래 "복붙 프롬프트"를 그대로 붙여넣고 가로/와이드로 요청.
  3. 비율 주의 — 정밀 2:1은 없음. 비율이 들쭉날쭉하면 "가로로 다시 그려줘"로 재요청 → 아래 규격화에서 크롭.
  4. 레퍼런스 첨부가 강점 — 기존 그림(style-anchors/stage_stylesheet.jpg 또는 첫 결과)을 업로드하고 "이 그림 스타일·구도로 ○○ 스테이지"라고 하면 톤·구도가 잘 유지됩니다. (이미지 편집·합성에 강함)
  5. 다운로드 → 아래 "규격화(공통)"으로.

👍 레퍼런스 이미지 첨부·편집(image-to-image)에 강함. AI Studio는 더 세밀한 제어 가능.

📋 복붙 프롬프트 (지피티·제미나이 공통)

밝고 깔끔한 카툰 2D 스테이지 배경, 외곽선 없음. 벨트스크롤·좌우 루프(양 끝이 오브젝트를 가로질러 반복), 가운데 집중·중앙 히어로 금지. 큼직한 덩어리 데포르메. 모든 오브젝트 밑동은 화면 약 48% 한 지면선에 정렬, 밀집은 상단 20~48%, 50% 아래로는 아무것도 없음(하단 절반은 빈 모래 플레이 레인, 긴 그림자만). 전경은 거의 비우고 큰 전경물(말뚝·유목·덤불) 금지 — 맨 아래 모서리 작은 잘린 실루엣만. 좌상단 단일 광원, 파스텔·낮은 콘트라스트(검정/순백/형광 금지). [바이옴: 화산]. 실사·3D·CGI 금지.

[바이옴: 화산] 부분만 초원·설원·사막·해안·항구 등으로 바꾸면 다른 스테이지가 됩니다.

📐 규격화 (공통 · 무설치)

Photopea 무료 웹(설치·가입 불필요): File → New로 2048 × 1024 캔버스 생성 → 받은 이미지를 드래그해 가로를 꽉 채우고 위(하늘)를 더 잘라 하단 플레이 레인을 살림 → File → Export as → PNG. 스테이지는 배경 제거 안 함(꽉 찬 배경).

⚠️ 화풍·규격을 사람이 직접 챙겨야 함. 반복·대량 작업엔 방법 A~B 권장.

🗺️ ② 필드 스테이지 만들기 (페인터리)

횡스크롤 전투 배경. 레퍼런스 실측 기반 규정. 상세는 STAGE-BACKGROUNDS.md.

화풍 · 형태

  • 밝고 깔끔한 카툰 일러스트 — 실사·3D 금지
  • 큼직한 덩어리 데포르메
  • 청록/녹색 룬 글리프 모티프
  • 일관된 단일 광원

구도 · 배치

  • 일반 스테이지 = 벨트스크롤 (가운데 집중 ❌)
  • 리듬감 — 크고작은 + 공백 (균등반복 ❌)
  • 좌우 무한 루프 — make-tileable.ps1로 마감
  • 오브젝트 상단 절반, 하단 절반 빈 레인 (50%)
  • 접합부 비일직선 — 밑동 엇갈림 + 풀·바위로 경계 깸 (선반 위 벽 ❌)
  • 유닛 레인 밝고 단순 — 캐릭터 가독성
  • HUD 세이프존 — 정중앙 16:9만 보임, 상단 모서리·하단 중앙 조용히

색 · 톤

  • 중간 채도(~36~42%) · 중간 콘트라스트 (물빠짐 ❌)
  • 근(近)백색 블로아웃 금지 — 끝단만 금지(검정/순백/형광)
  • 바이옴 적응형 + 가드레일 — 무드형은 더 어둡되 하단 레인 읽히게 / 눈·해안은 순백 블로아웃 ❌
  • 전경은 거의 비움 + 바닥 살짝 진하게 grounding (큰 전경물 ❌)

만드는 순서 · 규격

  1. 요청 — "화산지대 스테이지 만들어줘"
  2. 16:9로 생성 (GPT Image 2 권장)
  3. 2048×1024로 크롭 — crop-2048x1024.ps1 (보스 2600×1600·투기장 2560×1080은 crop-resize)
  4. 가로 루프 마감 — 일반 스테이지는 make-tileable.ps1로 좌우 이음매 제거
  5. 오브젝트 디테일만 바꿔 변형
  6. (선택) style-anchors/stage_stylesheet.jpg(중립 6바이옴 몽타주) 첨부로 공식 화풍에 더 똑같이 (image-to-image)

최신 규정 적용 예시 — 사막·화산·설원 (비일직선 접합 · 가로 루프 · 바이옴 가드레일)
