방법 A · Higgsfield (권장) 방법 B · Gemini 스크립트

빠른 시작 — 1분 설치

두 방법 모두 아래 2단계로 준비됩니다. 설치 스크립트는 시스템을 바꾸지 않는 안전한 점검형(파일 복사·안내만)이라 부담 없이 실행하세요.

1

저장소 클론

안에 README.md(사용법)와 PROMPTING.md(화풍 템플릿)가 들어 있습니다.

git clone https://github.com/sangdeoklee84/chrono.git
cd chrono
2
한 번만 실행

설치 스크립트 실행

필수 도구 점검 → .env 자동 생성 → 다음 단계 안내까지 한 번에 끝납니다. OS에 맞는 것만 실행하세요.

# Windows  (탐색기에서 install.ps1 우클릭 → "PowerShell로 실행" 도 가능)
./install.ps1

# macOS / Linux
bash install.sh

스크립트가 끝나면 화면 안내대로 방법 A(권장) 또는 방법 B로 진행합니다. ↓

방법 A · Higgsfield

권장 · 설치 불필요

Claude Code 안에서 한국어로 말하면 바로 생성됩니다. 배경 제거·업스케일까지 한 번에.

1
최초 1회만

Higgsfield 계정 연결

클론한 폴더엔 .mcp.json이 있어 이 폴더에서 Claude Code를 열면 higgsfield 서버가 자동 등록됩니다(허용 프롬프트가 뜨면 승인). 그다음 /mcp → higgsfield 선택 → 브라우저에서 로그인하고 Authorize(허용)까지 누르면 끝.

2

이미지 요청 — 그냥 말로

모델 선택·생성·대기·결과 확인을 Claude가 알아서 진행합니다.

마녀모자 쓴 할로윈 주점을 GPT Image 2로 만들어줘
3

배경 투명하게 (배치용 에셋이면 필수)

마을에 올릴 오브젝트는 투명 PNG로 컷아웃합니다.

방금 이미지 배경 투명하게 만들어줘
4

변형 세트 만들기

색·크기·시즌만 바꿔 시리즈로. 건물 형태까지 똑같이 두려면 원본을 레퍼런스로 넣어 “모자만 교체”로 요청하세요.

같은 건물로 모자 색만 4가지 다르게 만들어줘

방법 B · Gemini 스크립트

명령어로 대량 생성하거나 기존 이미지를 편집할 때. Node 18+ 필요.

1
최초 1회만

키만 채우기

설치 스크립트가 .env를 이미 만들어 뒀습니다. Google AI Studio에서 키를 발급(이미지 모델은 결제 활성화 필요)해 그 파일에 붙여넣기만 하면 됩니다.

# .env 파일을 열어: GEMINI_API_KEY=본인_키
2

새 이미지 생성

node --env-file=.env gemini/gen.mjs "프롬프트" out/결과.png
3

기존 이미지 편집 / 합성

node --env-file=.env gemini/gen.mjs "이 건물을 파괴 상태로" out/결과.png --in 원본.png

화풍 규칙 — 어느 방법이든 동일

만들 대상에 따라 화풍이 다릅니다. 자세한 프롬프트는 PROMPTING.md 참고.

마을 건물 · 소품

스텔라 킹덤 GUI · 셀셰이딩
  • 짙은 갈색 외곽선 #3A2A1E (검정 금지)
  • 면 단위 2~3단계 셀셰이딩 (그라데이션 X)
  • 따뜻한 파스텔 + 코랄 오렌지 포인트
  • 둥글고 통통, 30~45° 쿼터뷰
  • 투명 배경 + 바닥 드롭섀도우

인게임 스테이지 배경

크로노스피어 스테이지 · 페인터리
  • 외곽선 없는 디지털 페인팅 (셀셰이딩 금지)
  • 2:1 횡스크롤 + 원경/중경/전경 3레이어
  • 갓레이 · 대기 원근 · 바이옴별 단일 색
  • 하단은 비워 캐릭터 플레이 공간 확보
  • 2048×1024 (보스·던전은 더 큼)

스테이지 배경 규정 (페인터리)

인게임 횡스크롤 배경 제작 규정 — 기존 레퍼런스 실측 기반. 전체 상세는 저장소 STAGE-BACKGROUNDS.md 참고.

화풍 · 형태

  • 밝고 깔끔한 카툰 일러스트 — 실사·컨셉아트·3D 금지
  • 큼직한 덩어리 데포르메 (자잘함보다 큰 매스)
  • 청록/녹색 룬 글리프 시그니처 모티프
  • 일관된 단일 광원 — 모든 그림자 같은 방향

구도 · 배치

  • 일반 스테이지 = 벨트스크롤 (가운데 집중 ❌)
  • 리듬감 — 크고작은 + 중간 공백 (균등반복 ❌)
  • 좌우 타일 / 루프 가능
  • 오브젝트 상단 절반, 하단 절반은 빈 플레이 레인 (50% 중앙선)

색 · 톤

  • 파스텔 · 낮은 콘트라스트
  • 팔레트 양 끝단 금지 — 순수검정 / 순백 / 형광 ❌
  • 색 다양성 유지 (베이스 + 포인트·보색)
  • 전경만 콘트라스트 강조 — 원경흐림 → 중경파스텔 → 전경또렷 (3단 깊이)

규격 · 모델

  • 최종 2048×1024 (16:9 생성 → 2:1 크롭)
  • 스쿼시(왜곡) 금지 · 배경 제거 안 함
  • 모델: GPT Image 2 권장 (Nano Banana Pro는 현재 폴백됨)
  • 바이옴별 색 테마는 파스텔로 톤다운 적용

최신 규정 적용 예시 — 바이옴별 (드루이드 토템 유적 컨셉)

⛔ 절대 하지 말 것 — 보안

  • API 키를 코드·문서·git에 넣지 않기. 반드시 .env 또는 환경변수로만.
  • .env는 이미 .gitignore 처리됨 — 커밋되지 않습니다.
  • 생성 결과물 out/도 git에 올리지 않습니다. 각자 로컬에서 생성.
  • 키가 노출되면 즉시 폐기 후 재발급 (Google AI Studio).
